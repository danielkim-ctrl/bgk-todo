# bugfix/ — Playwright 버그 검증 스크린샷 저장소

## 폴더 네이밍 규칙

```
bugfix/
└── YYYYMMDD-{버그키워드}/
    ├── before/     ← 수정 전 재현 스크린샷
    └── after/      ← 수정 후 검증 스크린샷
```

### 버그키워드 규칙
- 한글 또는 영문, 공백 없이 하이픈(-) 연결
- 버그의 현상을 짧게 (2~4단어)
- 예시:
  - `20260506-팀배정-원복`
  - `20260506-프로젝트-사라짐`
  - `20260506-칸반-드래그-오류`
  - `20260506-modal-not-closing`

## 스크린샷 네이밍 규칙

```
before/
  01_재현상황설명.png     ← 번호 + 상황 설명
  02_오류발생화면.png

after/
  01_수정후동일시나리오.png
  02_정상동작확인.png
```

## 예시 구조

```
bugfix/
├── README.md
└── 20260506-팀배정-원복/
    ├── before/
    │   ├── 01_팀배정_클릭.png
    │   └── 02_원복된_화면.png
    └── after/
        ├── 01_팀배정_클릭.png
        └── 02_정상반영_확인.png
```

## 주의사항
- `before/`는 코드 수정 전에 반드시 캡처
- `after/`는 수정 후 동일한 시나리오로 재캡처
- 폴더 하나 = 버그 하나 (여러 버그 섞지 않기)
