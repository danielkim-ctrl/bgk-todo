## Quick Win 5가지 (임팩트 높고 작업량 적음)

1. **KPI 카드 "전체"/"지연" 클릭 동작 수정** — 빈 배열 대신 적절한 필터 전달
2. **리스트 셀에 hover 커서/배경 추가** — 편집 가능함을 시각적으로 안내
3. **칸반 카드 제목 말줄임 처리** — `text-overflow: ellipsis` 추가
4. **캘린더 팝업 viewport bounds 체크** — 화면 밖 넘침 방지
5. **대시보드 KPI 라벨 공백 제거** — "전 체 업 무" → "전체 업무"

## 뷰별 상세 분석

### 대시보드

#### [Critical] KPI 카드 클릭 미작동
- **화면**: 대시보드 상단 KPI 카드
- **상황**: "전체 업무" 또는 "지연" 카드를 클릭했을 때
- **현재 동작**: `onNavigate([])`로 빈 배열 전달 → 리스트 뷰로 이동하지만 필터 없이 전체 표시
- **기대 동작**: "전체"는 필터 없이 리스트 전체 표시, "지연"은 지연된 업무만 필터 적용
- **개선 제안**: "지연" 카드에 지연 업무 필터 로직 추가, "전체" 카드는 필터 초기화 동작으로 변경
- **영향 범위**: 대시보드 → 리스트 뷰 전환 흐름

#### [Medium] KPI 라벨 가독성 저하
- **화면**: 대시보드 KPI 카드
- **상황**: 카드 라벨이 "전 체 업 무", "진행 중", "완 료", "지 연"으로 글자 사이에 공백 삽입
- **현재 동작**: 한글 글자 사이에 인위적 공백이 들어가 읽기 불편
- **기대 동작**: "전체 업무", "진행중", "완료", "지연"으로 자연스러운 표기
- **개선 제안**: KPI 배열의 라벨 문자열에서 불필요한 공백 제거
- **영향 범위**: 대시보드 KPI 카드 4개

#### [Medium] 인원별/프로젝트별 탭 빈 상태 없음
- **화면**: 대시보드 하단 탭
- **상황**: 데이터가 없는 상태에서 인원별 또는 프로젝트별 탭 진입
- **현재 동작**: 빈 화면만 표시 (데일리 탭만 빈 상태 메시지 있음)
- **기대 동작**: "표시할 데이터가 없습니다" 안내 메시지
- **개선 제안**: 빈 상태 컴포넌트 추가
- **영향 범위**: 대시보드 2개 탭

#### [Medium] 긴 이름 말줄임 시 tooltip 없음
- **화면**: 대시보드 인원별/프로젝트별 통계
- **상황**: 긴 프로젝트명이나 담당자명이 말줄임(...)으로 표시될 때
- **현재 동작**: 잘린 텍스트의 전체 내용을 확인할 방법이 없음
- **기대 동작**: hover 시 tooltip으로 전체 이름 표시
- **개선 제안**: `title` 속성 추가
- **영향 범위**: 대시보드 테이블/그리드 전체

#### [Low] 색상 대비 부족
- **화면**: 대시보드 전체
- **상황**: `#94a3b8` 회색 텍스트가 흰 배경 위에 표시
- **현재 동작**: WCAG AA 기준 4.5:1 대비비 경계선
- **기대 동작**: 최소 4.5:1 이상 대비비 확보
- **개선 제안**: 보조 텍스트 색상을 `#64748b` 이상으로 상향
- **영향 범위**: 앱 전체 보조 텍스트

### 리스트 뷰

#### [High] 셀 편집 가능 여부 불명확
- **화면**: 리스트 뷰 테이블
- **상황**: 처음 사용하는 사용자가 테이블 셀을 볼 때
- **현재 동작**: 셀에 hover 효과, 커서 변경, 편집 아이콘 없음 — 클릭 가능한지 알 수 없음
- **기대 동작**: hover 시 배경 하이라이트 + `cursor: pointer` + 연필 아이콘
- **개선 제안**: `S.tdc`에 hover 스타일 추가, 편집 가능 셀에 시각적 단서 제공
- **영향 범위**: 리스트 뷰 전체 테이블

#### [High] 호버 액션 버튼 위치 불안정
- **화면**: 리스트 뷰 테이블 행
- **상황**: 행에 마우스를 올렸을 때 수정/삭제 버튼 표시
- **현재 동작**: `position: fixed`로 테이블 우측 외부에 표시 → 좁은 화면에서 viewport 밖으로 나감, 80ms debounce로 깜빡임
- **기대 동작**: 행 내부 마지막 셀에 인라인으로 표시
- **개선 제안**: fixed 대신 행 내부 absolute 배치, debounce 제거
- **영향 범위**: 리스트 뷰 모든 행

#### [Medium] 필수 항목 표시 없음
- **화면**: 업무 추가 섹션 (직접 입력 탭)
- **상황**: 새 행을 추가하고 입력할 때
- **현재 동작**: 어떤 필드가 필수인지 시각적 표시 없음 (업무명, 담당자가 필수)
- **기대 동작**: 필수 항목 헤더에 `*` 표시, 빈 상태로 저장 시 인라인 경고
- **개선 제안**: 테이블 헤더에 `*` 추가, 저장 시 빈 필수 필드 하이라이트
- **영향 범위**: 업무 추가 폼

#### [Medium] Shift+Click 범위 선택 미발견
- **화면**: 리스트 뷰 체크박스
- **상황**: 여러 업무를 선택하려 할 때
- **현재 동작**: Shift+Click 범위 선택 지원하지만 안내 없음
- **기대 동작**: 체크박스 hover 시 tooltip "Shift+클릭으로 범위 선택"
- **개선 제안**: tooltip 또는 하단 상태바에 안내 추가
- **영향 범위**: 리스트 뷰 일괄 작업

#### [Medium] 정렬 상태 인지 어려움
- **화면**: 리스트 뷰 정렬 버튼
- **상황**: 마감순/우선순위순 등으로 정렬 적용 후
- **현재 동작**: ▲/▼ 표시기 8px로 매우 작음
- **기대 동작**: 정렬된 컬럼 헤더 강조, 큰 화살표 아이콘
- **개선 제안**: 정렬 아이콘 크기 확대, 활성 정렬 버튼 배경 색상 추가
- **영향 범위**: 리스트 뷰 헤더

#### [Medium] 사이드바 "숨기기" vs "필터 해제" 혼동
- **화면**: 좌측 사이드바 필터
- **상황**: 프로젝트/담당자 옆 눈 아이콘 클릭
- **현재 동작**: "숨기기"는 해당 항목을 앱 전체에서 숨기고, "필터 해제"는 필터만 초기화. 두 개념의 차이가 불명확
- **기대 동작**: "숨기기"와 "필터"를 시각적으로 분리, tooltip으로 차이 설명
- **개선 제안**: 숨기기 아이콘에 tooltip "이 프로젝트를 모든 뷰에서 숨깁니다" 추가
- **영향 범위**: 사이드바 전체

### 캘린더 뷰

#### [Critical] 팝업 viewport 밖으로 넘침
- **화면**: 캘린더 이벤트 클릭 팝업, 빠른 추가 팝업
- **상황**: 화면 우측 또는 하단 근처의 이벤트를 클릭할 때
- **현재 동작**: `position: fixed`로 클릭 좌표에 표시하지만 bounds 체크 없음 → 296px/300px 팝업이 화면 밖으로 넘침
- **기대 동작**: `Math.min(x, window.innerWidth - 310)` 등으로 viewport 내에 유지
- **개선 제안**: 모든 fixed 팝업에 viewport bounds 체크 추가
- **영향 범위**: 캘린더 이벤트 팝업, 빠른 추가 팝업, 날짜별 상세 팝업

#### [High] 12개월 동시 렌더링 성능 이슈
- **화면**: 캘린더 월간 뷰
- **상황**: 월간 뷰 진입 시
- **현재 동작**: 1년 12개월 × 42셀 = 504+ DOM 노드 동시 렌더. 업무 많으면 2000+ 노드
- **기대 동작**: 현재 보이는 3~4개월만 렌더 (가상 스크롤 또는 레이지 로드)
- **개선 제안**: IntersectionObserver로 viewport 내 월만 렌더, 또는 3개월 단위 페이징
- **영향 범위**: 월간 뷰 전체

#### [Medium] 사이드바 접기 상태 초기화
- **화면**: 캘린더 우측 "나의 할일" 사이드바
- **상황**: 섹션을 접은 후 새로고침
- **현재 동작**: 모든 섹션이 다시 펼쳐짐 (상태 미저장)
- **기대 동작**: localStorage 또는 userSettings에 접기 상태 저장
- **개선 제안**: useUserSettings에 섹션 접기 상태 추가
- **영향 범위**: 캘린더 사이드바 4개 섹션

#### [Medium] 반복 인스턴스 시각적 구분 부족
- **화면**: 캘린더 이벤트 칩
- **상황**: 반복 업무의 인스턴스가 캘린더에 표시될 때
- **현재 동작**: 반복 아이콘(ArrowPathIcon)만 추가 — 일반 이벤트와 거의 동일
- **기대 동작**: 반복 인스턴스 칩에 배경색 차이 또는 점선 테두리로 구분
- **개선 제안**: `evStyle`에서 `_instance` 여부에 따라 배경 opacity 또는 border 변경
- **영향 범위**: 캘린더 모든 뷰의 반복 이벤트

#### [Medium] 빈 날짜 클릭 → 빠른 추가 미발견
- **화면**: 캘린더 일간/주간/커스텀 뷰
- **상황**: 빈 시간대를 보고 있을 때
- **현재 동작**: 빈 셀 클릭 시 빠른 추가 팝업이 나오지만 시각적 힌트 없음
- **기대 동작**: hover 시 연한 "+" 아이콘이나 점선 테두리 표시
- **개선 제안**: 빈 셀 hover 시 `background: #f8fafc` + "+" 아이콘 미리보기
- **영향 범위**: 캘린더 빈 시간대 전체

#### [Low] browser confirm() 사용
- **화면**: 캘린더 사이드바 업무 삭제
- **상황**: 업무 삭제 시
- **현재 동작**: 브라우저 기본 `confirm()` 다이얼로그 표시
- **기대 동작**: 앱 디자인과 일치하는 커스텀 확인 모달
- **개선 제안**: Modal 컴포넌트로 삭제 확인 UI 통일
- **영향 범위**: 삭제 확인이 필요한 모든 곳

### 칸반 뷰

#### [Critical] 모바일/태블릿에서 사용 불가
- **화면**: 칸반 뷰 전체
- **상황**: 768px 미만 화면에서 접속
- **현재 동작**: `gridTemplateColumns: "repeat(4,1fr)"` 고정 — 4개 컬럼이 20%씩 쪼개져 텍스트 안 보임
- **기대 동작**: 반응형 그리드 — 768px 미만 시 2열 또는 1열 + 가로 스크롤
- **개선 제안**: `window.innerWidth` 기반 분기 또는 CSS `minmax()` 사용
- **영향 범위**: 칸반 뷰 전체 레이아웃

#### [High] 긴 업무명 카드 오버플로우
- **화면**: 칸반 카드
- **상황**: 긴 업무명이 입력된 카드
- **현재 동작**: 텍스트 줄바꿈으로 카드 높이가 불규칙하게 증가, `text-overflow: ellipsis` 없음
- **기대 동작**: 2줄까지 표시 후 말줄임, 일정한 카드 높이 유지
- **개선 제안**: `display: -webkit-box; -webkit-line-clamp: 2; overflow: hidden` 적용
- **영향 범위**: 칸반 모든 카드

#### [Medium] "+" 버튼 발견 어려움
- **화면**: 칸반 컬럼 헤더
- **상황**: 특정 상태에 업무를 추가하려 할 때
- **현재 동작**: "+" 버튼 `opacity:.75`, `fontWeight:300`, 색상도 연함
- **기대 동작**: hover 시 `opacity:1` + 배경 하이라이트
- **개선 제안**: hover 스타일 추가, 기본 opacity 상향
- **영향 범위**: 칸반 4개 컬럼

#### [Medium] 드롭 위치 표시선 인지 어려움
- **화면**: 칸반 카드 드래그 중
- **상황**: 카드를 드래그하여 다른 위치에 놓으려 할 때
- **현재 동작**: 3px 파란 실선 — 빠른 이동 중 눈에 잘 안 띔
- **기대 동작**: 4px 이상 + 글로우 효과 또는 점선
- **개선 제안**: 높이 4px + `box-shadow: 0 0 6px #2563eb55` 추가
- **영향 범위**: 칸반 드래그앤드롭

#### [Low] 카드 클릭 가능 여부 불명확
- **화면**: 칸반 카드
- **상황**: 카드를 보고 있을 때
- **현재 동작**: `cursor:grab`만 설정 — 드래그만 가능해 보임
- **기대 동작**: 클릭하면 상세 모달이 열리는데, 이를 알려주는 `cursor:pointer` 필요
- **개선 제안**: hover 시 카드 그림자 강화 + cursor 변경으로 클릭 가능함을 암시
- **영향 범위**: 칸반 모든 카드
