Sticky 노트 UI를 적용해줘

## 목적
- 현재 스티키노트(포스트잇)로 할 일을 관리하는 사람들도 자연스럽게 이 투두앱을 사용할 수 있도록 한다.
- 기존 스티키노트 사용자에게 친숙한 경험을 제공하여 디지털 전환의 진입 장벽을 낮춘다.

## UI 원칙
- 기본 UI는 스티키노트(포스트잇) 스타일로 구성한다.
- 노트 카드는 포스트잇처럼 보여야 한다 (노란색 계열 배경, 살짝 기울어진 느낌, 그림자 등).
- 할 일 항목은 마치 포스트잇에 손으로 쓴 것처럼 자연스럽고 가벼운 느낌을 준다.
- 카드 추가/삭제는 포스트잇을 붙이고 떼는 인터랙션과 유사하게 표현한다.

## 업무 배정 규칙
- 새로 생성되는 업무는 기본적으로 현재 로그인한 사용자의 이름으로 자동 배정된다.
- 사용자는 필요 시 담당자를 변경할 수 있다.

## 업무 필드 구성
- 각 업무 카드에는 다음 필드가 포함된다: 프로젝트, 마감기한, 우선순위, 상태, 담당자.
- 스티키노트 카드의 상단 영역에 작은 폰트로 메타 정보(프로젝트, 마감기한, 우선순위, 상태)를 표기한다.
- 메타 정보는 카드 본문(업무 내용)보다 시각적으로 작고 흐리게 처리하여 스티키노트 느낌을 해치지 않도록 한다.
- 카드 하단에는 담당자 이름을 표기한다.

## 스티키노트 사용자 이질감 해소 아이디어

### 색상
- 카드 색상을 사용자가 직접 선택할 수 있다 (노랑/초록/분홍/파랑/보라 등 포스트잇 실제 색상 팔레트 기준).
- 프로젝트별로 기본 색상이 자동 배정되어, 실제로 프로젝트마다 다른 색 포스트잇을 쓰는 습관을 반영한다.

### 배치 & 레이아웃
- 카드들이 격자(grid)가 아니라 자유롭게 흩뿌려진 것처럼 보이도록 각 카드에 미세한 랜덤 rotation(-2도 ~ +2도)을 적용한다.
- 칸반 뷰에서 카드가 세로로 쌓일 때 살짝 겹쳐진 것처럼 그림자를 활용해 깊이감을 준다.

### 입력 경험
- 새 카드를 추가할 때 "포스트잇을 붙이는" 느낌의 짧은 애니메이션(위에서 툭 떨어지거나 살짝 튀는 효과)을 적용한다.
- 카드 삭제 시 포스트잇을 떼어내는 느낌의 슬라이드-업 애니메이션을 적용한다.
- 업무명 입력 필드는 줄 간격이 넓고 폰트가 손글씨에 가까운 느낌(예: Nanum Pen Script 등)을 옵션으로 제공한다.

### 텍스트 & 폰트
- 카드 본문 폰트를 약간 캐주얼한 느낌으로 설정하고, 메타 정보(상태/날짜 등)만 고딕 계열로 구분하여 "포스트잇에 메모한 느낌"을 준다.
- 업무명이 짧을수록 폰트 크기가 자동으로 커져서, 포스트잇에 크게 써놓은 것처럼 보이게 한다 (최대 3단계 크기 조절).

### 물리적 질감
- 카드 배경에 종이 질감 노이즈(subtle paper texture)를 CSS로 적용해 실제 포스트잇처럼 보이게 한다.
- 카드 상단에 접착 부분을 나타내는 약간 어두운 띠(top strip)를 시각적으로 표현한다.

### 완료 처리
- 업무를 완료 상태로 변경하면 카드에 대각선 줄긋기(취소선) 효과가 생기고 색이 바래는 애니메이션을 적용한다.
- 실제 포스트잇을 구겨서 버리는 것처럼 완료된 카드는 "접힌" 느낌의 시각 처리를 옵션으로 제공한다.

### 공간 감각
- 뷰를 "벽(Board)" 개념으로 표현하여 사용자가 실제 벽이나 유리창에 포스트잇을 붙이는 공간에 있다는 감각을 준다.
- 배경을 코르크보드, 화이트보드, 또는 유리창 텍스처 중 선택할 수 있게 한다.
