## 용어 설명

이 문서에서 자주 등장하는 개발 용어를 먼저 정리합니다.

| 용어 | 뜻 |
|------|-----|
| **XSS** | 악성 코드 삽입 공격. 메모에 특수 코드를 넣으면 다른 팀원 화면에서 실행될 수 있는 보안 문제 |
| **dangerouslySetInnerHTML** | HTML 코드를 그대로 화면에 표시하는 기능. 악성 코드가 섞여 있으면 위험 |
| **DOMPurify** | HTML에서 위험한 코드를 자동으로 걸러주는 보안 도구 |
| **CTA (Call To Action)** | "이걸 클릭하세요"라고 사용자를 유도하는 버튼이나 링크 |
| **sticky** | 스크롤해도 화면 상단에 계속 붙어있는 고정 효과 |
| **viewport** | 지금 화면에 보이는 영역 |
| **hover** | 마우스를 올렸을 때 나타나는 시각 효과 |
| **포커스 트랩** | 팝업이 열렸을 때 키보드(Tab키)가 팝업 안에서만 움직이게 가두는 기능 |
| **WCAG** | 웹 접근성 국제 표준. 장애인도 웹을 쓸 수 있도록 하는 규칙 |
| **aria 속성** | 화면 읽어주는 프로그램(스크린리더)이 이해할 수 있도록 붙이는 설명 태그 |
| **inline 편집** | 별도 화면 없이 목록에서 바로 글자를 수정하는 방식 |
| **contentEditable** | 웹 화면의 텍스트를 직접 수정할 수 있게 만드는 기능 |
| **execCommand** | 글자 굵기, 기울기 등 서식을 적용하는 오래된 방식 (곧 사라질 예정) |
| **deprecated** | 더 이상 권장하지 않는 기능. 언젠가 삭제될 수 있음 |
| **portal** | 화면의 특정 위치가 아닌 최상위 위치에 팝업을 띄우는 기법 |
| **하드코딩** | 값을 한곳에서 관리하지 않고 여기저기 직접 적어넣는 것 (수정 시 빠뜨리기 쉬움) |
| **반응형** | 화면 크기(PC, 태블릿, 모바일)에 따라 자동으로 레이아웃이 바뀌는 것 |
| **터치 타깃** | 손가락으로 눌러야 하는 버튼의 크기. 너무 작으면 잘못 누르기 쉬움 |
| **키프레임** | 움직임(애니메이션)을 정의하는 코드. 미정의 시 애니메이션이 안 나옴 |
| **서브픽셀** | 화면에서 1픽셀보다 작은 단위. 미세한 어긋남의 원인 |
| **gap** | 버튼이나 요소 사이의 간격 |
| **padding** | 요소 안쪽 여백 (글자와 테두리 사이 공간) |
| **opacity** | 투명도. 0은 완전 투명, 1은 완전 불투명 |
| **calc()** | 화면 크기에 맞춰 자동 계산하는 수식 |
| **colgroup** | 표(테이블)에서 열의 너비를 미리 지정하는 방법 |

## Critical (즉시 수정) — 2건

### C-1. 보안 취약점 — 메모에 악성 코드를 넣으면 다른 팀원에게 실행됨

- **위치**: 상세보기(DetailView), 캘린더 사이드바, 메모뷰
- **현재 상태**: 사용자가 입력한 HTML(서식이 적용된 텍스트)을 아무 검사 없이 그대로 화면에 표시
- **문제**: 누군가 메모에 `<script>alert(1)</script>` 같은 악성 코드를 넣으면, 그 업무를 보는 모든 팀원의 브라우저에서 해당 코드가 실행됨
- **해결 방법**: DOMPurify(보안 필터 도구)를 설치해서, 화면에 표시하기 전에 위험한 코드를 자동으로 제거
- **영향 범위**: 상세보기, 캘린더 사이드바, 메모뷰의 텍스트 편집 영역

### C-2. "첫 업무 추가하기" 버튼이 작동하지 않음

- **위치**: 리스트뷰에서 업무가 하나도 없을 때 나오는 화면
- **현재 상태**: 버튼 클릭 시 "direct"라는 잘못된 값을 전달 (실제로는 "manual"이어야 함)
- **문제**: "첫 업무 추가하기" 버튼을 눌러도 아무 일이 안 일어남
- **해결 방법**: `"direct"` → `"manual"`로 한 글자만 수정
- **영향 범위**: 리스트뷰 빈 화면의 안내 버튼

## High (핵심 기능에 영향) — 14건

### H-1. 탭 순서가 실제 사용 빈도와 맞지 않음

- **위치**: 상단 탭 메뉴 (대시보드 → 리스트 → 캘린더 → 칸반)
- **문제**: 가장 자주 쓰는 "리스트"가 두 번째에 있음. 첫 번째여야 자연스러움
- **제안**: 리스트 → 칸반 → 캘린더 → 대시보드 순으로 변경

### H-2. 로그아웃 버튼을 실수로 누르면 바로 로그아웃됨

- **위치**: 헤더 우측 사용자 영역
- **현재 상태**: 글자 크기가 매우 작고(10px), 확인 없이 즉시 로그아웃
- **문제**: 실수로 한 번 탭하면 바로 로그아웃. 되돌릴 수 없음
- **제안**: "정말 로그아웃하시겠습니까?" 확인 대화 추가, 버튼 크기도 키움

### H-3. 검색창에 긴 글자를 입력하면 X(지우기) 버튼과 글자가 겹침

- **위치**: 사이드바 검색창
- **현재 상태**: 검색어가 길어지면 오른쪽 끝 글자가 X 버튼 아래에 숨겨짐
- **제안**: 글자가 표시되는 영역의 오른쪽 여백을 더 넓혀서 겹치지 않게 함

### H-4. 우선순위/상태 필터에 불필요한 "미배정" 옵션이 표시됨

- **위치**: 사이드바 필터 영역
- **현재 상태**: 프로젝트, 담당자, 우선순위, 상태 모두에 "미배정" 옵션이 있음
- **문제**: 우선순위와 상태는 항상 값이 있으므로 "미배정"이 의미 없음
- **제안**: 프로젝트와 담당자에만 "미배정" 옵션을 표시

### H-5. 고정 헤더 영역이 화면을 너무 많이 차지함

- **위치**: 리스트뷰 상단 (업무 개수 + 필터 표시 + 정렬 바 + 표 머리글이 모두 고정)
- **문제**: 작은 화면(노트북 등)에서 고정 영역이 200px 이상 차지 → 실제 업무 목록을 볼 공간이 부족
- **제안**: 필터 표시 영역은 가로 스크롤로 바꾸거나, 고정에서 제외

### H-6. 표 머리글과 본문의 열 너비가 미세하게 어긋남

- **위치**: 리스트뷰 (머리글용 표와 본문용 표가 별도로 존재)
- **문제**: 두 개의 표가 따로 있어서 열 너비가 아주 살짝 다르게 렌더링될 수 있음
- **제안**: 하나의 표로 합치거나, 열 너비를 고정 값으로 맞춤

### H-7. 마감기한 셀 클릭 시 두 가지 동작이 동시에 실행됨

- **위치**: 리스트뷰에서 마감기한 칸 클릭
- **현재 상태**: 셀 편집 모드 + 날짜 선택 팝업이 동시에 활성화
- **문제**: 셀이 파란 테두리로 강조되면서 동시에 팝업도 뜸 → 시각적으로 혼란
- **제안**: 마감기한은 팝업만 열리도록 변경

### H-8. 필터 표시(칩)가 공통 부품을 사용하지 않고 직접 만들어져 있음

- **위치**: 리스트뷰 필터 영역
- **현재 상태**: 필터를 표시하는 태그가 이미 만들어둔 Chip(공통 부품)을 안 쓰고 따로 구현
- **문제**: 다른 곳의 필터 표시와 모양이 달라질 수 있음
- **제안**: 기존에 만들어둔 Chip 부품을 활용

### H-9. 캘린더와 칸반의 필터 동작이 서로 다름

- **위치**: 캘린더 (하나만 선택 가능) vs 칸반 (여러 개 동시 선택 가능)
- **문제**: 같은 "필터" 기능인데 화면마다 동작이 달라 혼란
- **제안**: 모든 화면에서 여러 개 동시 선택 방식으로 통일

### H-10. 화면 가장자리에서 팝업이 화면 밖으로 넘침

- **위치**: 캘린더에서 날짜나 업무를 클릭할 때 나오는 팝업
- **현재 상태**: 클릭한 위치에 그대로 팝업이 열림
- **문제**: 화면 오른쪽 끝이나 아래쪽 끝에서 클릭하면 팝업이 잘리거나 화면 밖으로 나감
- **제안**: 화면 경계를 넘지 않도록 위치를 자동으로 조정

### H-11. 팝업(모달) 열렸을 때 키보드 Tab이 배경 요소로 빠져나감

- **위치**: 모든 팝업/모달
- **현재 상태**: 팝업이 열려 있는데도 Tab 키를 누르면 뒤에 있는 버튼이나 입력칸으로 이동
- **제안**: 팝업이 열려 있을 때는 Tab이 팝업 안에서만 돌도록 제한

### H-12. 대시보드 통계 카드가 항상 4열 고정

- **위치**: 대시보드 상단 KPI 카드 (전체/진행중/완료/지연)
- **문제**: 화면이 좁아도 4개가 한 줄에 나와서 글자가 줄바꿈되거나 읽기 어려움
- **제안**: 화면 크기에 따라 2열 또는 1열로 자동 조정

### H-13. 캘린더에서 반복 업무의 투명도 설정에 오류 가능성

- **위치**: 캘린더 뷰의 반복 업무 표시 부분 (여러 곳)
- **현재 상태**: `t._instance?.28` — 코드가 "투명도 0.28"이 아니라 "속성 28번"으로 해석될 수 있음
- **문제**: 반복 업무 카드의 투명도가 의도와 다르게 표시될 수 있음
- **제안**: `t._instance ? 0.28 : 0.4`로 명확하게 수정

### H-14. 대시보드 상태 색상이 사용자 설정을 무시함

- **위치**: 대시보드의 상태별 진행 바
- **현재 상태**: 색상이 코드에 직접 적혀 있어서, 설정에서 변경한 색상이 반영되지 않음
- **제안**: 설정에서 지정한 색상을 사용하도록 변경

## Medium (불편하지만 사용은 가능) — 59건

### 헤더/네비 영역 (6건)

|---|------|---------|
| M-1 | 로고 클릭 시 페이지 새로고침 → 보고 있던 화면 상태가 모두 초기화 | 새로고침 대신 리스트뷰로 이동 + 맨 위로 스크롤 |

| M-6 | 헤더의 구분선 간격이 곳마다 다름 | 동일한 간격으로 통일 |
