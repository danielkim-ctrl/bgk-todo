# BGK TODO 시스템 기획 평가서

> 작성 기준: planning.md 15년차 시니어 기획자 관점
> 평가 대상: BGK 팀 TODO 통합관리 시스템 (현재 버전)
> 평가 방식: 기능 현황 파악 → 사용자 관점 검토 → 리스크 분석 → 우선순위별 개선 제안

## 총평

이 시스템은 **기능적 완성도는 높지만, 사용성과 신뢰성 측면에서 구조적 보완이 필요한 단계**입니다.
멀티뷰(리스트/칸반/캘린더/대시보드), AI 업무 추출, 실시간 Firestore 동기화까지 갖춘 수준 높은 팀 툴이지만, 사용자 입장에서 "처음 써도 직관적으로 이해할 수 있는가"에 대한 답이 아직 부족합니다.

**잘하고 있는 것 (유지할 것)**
- 뷰 전환의 다양성 (리스트/칸반/캘린더/메모/대시보드)
- AI 기반 업무 자동 추출 (Anthropic Claude API 연동)
- 실시간 팀 동기화 (Firestore)
- 50단계 실행 취소(Undo) 지원
- 사용자별 즐겨찾기, 필터 설정 개인화

**즉시 고쳐야 할 것 (Critical)**
- 처음 진입한 사용자가 무엇을 해야 할지 모름 (온보딩 부재)
- 데이터가 없을 때 화면이 비어 있기만 함 (Empty State 미설계)
- 삭제/완료 등 되돌릴 수 없는 액션에 대한 보호 장치 부족
- 입력 오류 시 사용자에게 명확한 피드백 없음

## 1. 정보 구조(IA) 평가

### 현황

```
헤더 (로고 / 타이틀 / 설정버튼들 / 사용자)
  └── 탭 네비게이션 (대시보드 / 칸반 / 리스트 / 캘린더)
        └── 사이드바(필터) + 메인 콘텐츠 영역
```

### 문제점

#### IA-1. 탭 순서가 사용 빈도와 맞지 않는다
- 현재 순서: 대시보드 → 칸반 → **리스트** → 캘린더
- 실제로 가장 자주 쓰이는 화면은 **리스트** (업무 확인/추가/수정의 핵심 뷰)
- 대시보드는 주간/월간 확인용으로 사용 빈도가 낮음
- **→ 리스트를 첫 번째 탭으로 이동 권장**

#### IA-2. 설정 기능이 헤더에 분산되어 있다
- "프로젝트", "설정", "되돌리기", "다시 실행"이 같은 헤더 영역에 혼재
- 관리 기능(프로젝트/설정)과 작업 기능(되돌리기/다시 실행)이 같은 레벨로 보임
- **→ 관리 기능은 우측 설정 아이콘 하위로 통합, 되돌리기는 단축키로만 운영 고려**

#### IA-3. "나의 할 일" 사이드바가 캘린더 뷰 전용으로만 존재한다
- 리스트/칸반 뷰에서는 좌측 사이드바가 "필터" 역할
- 캘린더 뷰에서만 "나의 할 일" 사이드바가 우측에 별도 존재
- 같은 앱인데 뷰마다 사이드바 개념과 위치가 달라 혼란 발생
- **→ "나의 할 일"을 독립적인 상시 접근 패널로 분리하거나, 전체 뷰 공통으로 통합**

## 2. 사용자 흐름(User Flow) 평가

### Flow-1. 신규 사용자 진입 흐름 ❌

**현재 흐름:**
로그인 → (아무 안내 없이) 빈 리스트 또는 샘플 데이터 화면

**문제:**
- 처음 진입한 팀원이 "업무를 어떻게 추가하나요?"를 물어봐야 알 수 있음
- AI 입력, 캘린더, 칸반 등 기능이 있는지조차 모를 수 있음
- 샘플 데이터가 있어도 "이것이 샘플인가, 실제 데이터인가?" 구분이 불가

**리스크:**
- 첫 1주 내 이탈 가능성 높음
- "그냥 엑셀 쓰자"로 회귀할 수 있음

**권장 개선:**
```
신규 사용자 첫 로그인 시 →
  1. 핵심 기능 3가지 소개 툴팁 (업무 추가 / 뷰 전환 / 필터)
  2. 샘플 데이터에 "샘플 데이터입니다 — 삭제하려면 여기를 클릭" 배너 표시
  3. 빈 상태 시 "첫 업무를 추가해보세요 →" 행동 유도 버튼
```

---

### Flow-2. 업무 추가 흐름 ⚠️

**현재 흐름:**
직접 입력 탭 → 빈 행 입력 → 저장

**문제:**
- "+ 직접 입력" 탭을 클릭하면 입력 폼이 나오는데, 어디에 포커스가 가는지 즉시 알기 어려움
- 필수 항목(업무명, 담당자)이 표시되지 않아 어떤 것을 꼭 입력해야 하는지 모름
- 저장 후 방금 추가된 업무가 어디 있는지 시각적으로 강조되지 않음 (목록 어딘가에 섞임)

**권장 개선:**
```
- 입력 행 첫 필드(업무명)에 자동 포커스
- 필수 항목에 * 표시 또는 "필수" 레이블
- 저장 후 해당 행 하이라이트 1~2초 유지
```

---

### Flow-3. 업무 완료 처리 흐름 ⚠️

**현재 흐름 (리스트 뷰):**
상태 드롭다운 → "완료" 선택

**현재 흐름 (사이드바):**
체크박스 클릭 → 완료 표시

**문제:**
- 완료 처리 방법이 뷰마다 다름 (체크박스 / 드롭다운 / 칸반 드래그)
- "완료된 업무를 어디서 확인하나?" → 리스트 하단에 숨겨져 있거나, 필터로 찾아야 함
- 완료 취소(되돌리기)가 어디서 가능한지 불분명

**권장 개선:**
```
- 완료 처리 후 "완료 취소" 토스트 알림에 버튼 추가 (3초 내 실행 취소)
- 완료 목록 접기/펼치기 UI를 모든 뷰에서 일관되게 제공
```

---

### Flow-4. 필터 사용 흐름 ⚠️

**현재 흐름:**
사이드바에서 필터 클릭 → 목록 즉시 변경

**문제:**
- 필터가 적용됐는지 메인 화면에서 한 눈에 확인이 어려움
- 여러 필터를 동시에 켰을 때 "어떤 조건으로 보고 있는가?"가 불투명
- 필터 초기화 버튼이 사이드바 최하단에 위치 (스크롤 해야 발견 가능)

**권장 개선:**
```
- 활성 필터를 헤더 또는 리스트 상단에 항상 태그(Chip)로 표시
- 필터 초기화 버튼을 활성 필터 태그 옆에 항상 노출
- "N개 필터 적용 중" 요약 표시
```

## 4. 리스크 분석

### Risk-1. 데이터 신뢰성 리스크 🔴 (높음)

**문제:**
- 여러 사람이 동시에 같은 업무를 수정하면 Firestore 덮어쓰기 발생
- 현재 "마지막 저장이 이김(Last Write Wins)" 방식
- 누군가의 수정이 조용히 사라질 수 있음

**리스크:** 중요한 업무 수정 내역 유실 → 팀 신뢰도 저하

**권장 개선:**
- 충돌 감지 시 "다른 사람이 수정했습니다. 어떤 버전을 유지할까요?" 알림
- 업무별 수정 이력 최소 5회 보관

---

### Risk-2. 권한 없는 삭제 리스크 🔴 (높음)

**문제:**
- 현재 모든 사용자가 모든 업무를 삭제/수정 가능
- 타인이 담당자인 업무도 삭제 가능
- 실수로 전체 선택 후 Delete 키 → 전체 삭제

**리스크:** 실수 또는 악의적 조작으로 팀 전체 데이터 유실

**권장 개선 (단기):**
- 삭제 재확인 UI를 `confirm()` 팝업 → 직접 "삭제"를 타이핑하는 방식으로 강화
- 삭제된 업무 30일 보관 (휴지통 기능)

**권장 개선 (중기):**
- 역할 기반 권한 적용 (본인 업무만 삭제/타인 업무는 읽기 전용)

---

### Risk-3. 성능 리스크 🟡 (중간)

**문제:**
- 업무 전체를 한 번에 메모리에 로드 (페이지네이션 없음)
- 캘린더 12개월 전체 동시 렌더링
- 필터 변경마다 전체 배열 재계산

**리스크:** 업무가 200건 이상 될 때 리스트 스크롤 버벅임, 캘린더 진입 지연

**권장 개선:**
- 리스트: 100건 단위 페이지네이션 또는 무한 스크롤
- 캘린더: 현재 월 ± 1개월만 렌더링

---

### Risk-4. 모바일 접근 리스크 🟡 (중간)

**문제:**
- 반응형 레이아웃 미적용
- 태블릿/모바일에서 사이드바 + 테이블이 겹치거나 넘침
- 터치 드래그 앤 드롭 미검증

**리스크:** 회의 중 모바일로 업무 확인이 불가능 → 앱 사용 이탈

**권장 개선:**
- 768px 미만: 사이드바 숨기고 상단 필터 버튼으로 대체
- 모바일 전용 하단 내비게이션 바 추가
- 터치 인터랙션 검증 (특히 칸반, 캘린더)

---

### Risk-5. 온보딩 리스크 🟡 (중간)

**문제:**
- 신규 팀원이 스스로 사용법을 익힐 방법이 없음
- AI 입력, 칸반, 반복 업무 등 숨겨진 기능이 많음
- API 키 설정을 몰라서 AI 기능 못 쓰는 경우 발생

**리스크:** 강력한 기능이 있어도 활용률이 낮아 ROI 저하

## 5. 누락된 핵심 기능 (우선순위 순)

### 🔴 즉시 필요

| # | 기능 | 이유 |
|---|------|------|
| 1 | **마감 임박 알림** | 마감일이 지난 업무를 앱에 들어가야만 알 수 있음 → 브라우저 알림 또는 이메일 |
| 2 | **휴지통 / 삭제 이력** | 실수로 삭제 시 복구 불가 → 30일 보관 |
| 3 | **빈 상태(Empty State) 설계** | 업무 없을 때 행동 유도 없음 |

### 🟡 단기 필요 (1~2개월)

| # | 기능 | 이유 |
|---|------|------|
| 4 | **반복 업무 인스턴스 생성** | 반복 설정은 있는데 캘린더에 미래 일정이 안 보임 |
| 5 | **업무 댓글 / 메모 이력** | 업무 진행 중 소통 내역이 남지 않음 |
| 6 | **대시보드 기간 필터** | 주간/월간 성과 확인 불가 |

### Detail-2. 단축키 안내 부재

## 7. 개선 로드맵 (권장 순서)

### Phase 1 — 신뢰성 확보 (즉시, 1~2주)
1. 빈 상태 UI + 행동 유도 버튼 추가
2. 삭제된 업무 휴지통 기능 (30일 보관)
3. 필터 적용 상태 항상 가시화 (필터 태그 + 개수)
4. 오류 메시지 사용자 친화적으로 개선
5. 단축키 도움말 팝업 (`?` 키)

### Phase 2 — 사용성 개선 (단기, 1개월)
1. 리스트 탭을 첫 번째로 이동
2. 마감 임박/초과 업무 알림 (브라우저 Notification API)
3. AI 결과 확인 후 즉시 실행 취소 토스트
4. 반복 업무 캘린더 인스턴스 표시
5. 대시보드 기간 필터 + 클릭 연결

### Phase 3 — 확장성 준비 (중기, 2~3개월)
1. 모바일 반응형 레이아웃
2. 역할 기반 권한 (읽기/쓰기/관리자)
3. 업무 댓글/메모 이력
4. CSV 내보내기
5. 리스트 100건 이상 페이지네이션
