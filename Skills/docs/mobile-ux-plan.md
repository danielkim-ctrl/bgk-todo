# BGK Todo — 모바일 UX 기획안

> 기준: UXdesigner.md 모바일 설계 원칙
> 대상: 현재 웹 앱(React + Firebase)을 반응형으로 확장
> 브레이크포인트: **768px 이하 = 모바일**, 769px 이상 = 데스크톱(현재 유지)

---

## 1. 전체 레이아웃 전환

### 현재 (웹)
```
┌──────────────────────────────────────────┐
│ 헤더 (52px, #172f5a)                      │ ← sticky
├──────────────────────────────────────────┤
│ 탭 네비 (40px)                            │ ← sticky, top:52px
├──────────────────────────────────────────┤
│ [사이드바 240px] [메인 콘텐츠 영역]        │
│                                          │
└──────────────────────────────────────────┘
```

### 변경 후 (모바일)
```
┌──────────────────────────────────────────┐
│ 헤더 (48px)  [≡] [타이틀]  [검색] [+]   │ ← sticky, top:0
├──────────────────────────────────────────┤
│                                          │
│ 메인 콘텐츠 영역 (전체 너비)              │
│                                          │
│                                 [ + FAB] │ ← 업무 추가 플로팅 버튼
├──────────────────────────────────────────┤
│ [대시] [리스트] [칸반] [캘린더] [더보기]  │ ← 하단 탭 바 (56px)
└──────────────────────────────────────────┘
```

### 핵심 변경 사항

| 항목 | 웹 | 모바일 |
|------|-----|--------|
| 상단 탭 네비 | 헤더 아래 40px | **제거** (하단 탭으로 이동) |
| 하단 탭 바 | 없음 | **신규** 56px 고정 |
| 사이드바 | 항상 표시 | **숨김** → 햄버거 → Drawer |
| 헤더 우측 버튼 | 설정·휴지통·단축키·사용자 | 검색 아이콘 + [+] 버튼만 노출 |
| 헤더 좌측 | 로고+제목 | 햄버거 + 타이틀 |
| 메인 padding | 16px 24px | 12px 16px |

## 2. 화면별 기획

---

### A. 헤더 (모바일)

```
┌─────────────────────────────────────────┐
│ [≡]   BGK Todo          [🔍]  [설정...] │
└─────────────────────────────────────────┘
```

**구성 요소:**
- **[≡] 햄버거**: 사이드바 Drawer 열기, 44×44px 탭 영역
- **타이틀**: 현재 뷰 이름 표시 ("리스트", "칸반" 등)
- **[🔍] 검색**: 탭 시 검색 바 전체 너비로 확장 (슬라이드 애니메이션)
- **[···] 더보기**: 설정·휴지통·단축키·사용자 → 하단 액션 시트로

**현재 헤더에서 제거되는 것:**
- 설정·되돌리기·다시실행·단축키 아이콘 → 더보기 액션 시트 내부로
- 사용자 아바타 → Drawer 상단에 배치

---

### B. 하단 탭 바 (Bottom Tab Bar) — 신규

```
┌──────┬──────┬──────┬──────┬──────┐
│  📊  │  ☰  │  ⊞  │  📅  │ ···  │
│대시보│리스트│ 칸반 │캘린더│더보기│
└──────┴──────┴──────┴──────┴──────┘
```

**규칙:**
- 높이 56px + `safe-area-inset-bottom` (iPhone 홈 인디케이터 대응)
- 현재 탭: 아이콘 filled(solid) + 파란색 `#2563eb` + 라벨 표시
- 비활성 탭: 아이콘 outline + 회색 `#94a3b8`
- 지연 업무 수 있으면 리스트 탭 아이콘에 빨간 배지
- "더보기": 대시보드 뷰 (현재 사용 빈도 낮음 → 하단으로 이동 고려)

**탭 순서 (모바일 최적화):**
> 리스트 → 칸반 → 캘린더 → 대시보드 → 더보기
> (웹과 달리 리스트를 첫 번째로 — 가장 자주 쓰는 뷰)

### C. 사이드바 → Drawer (모바일)

**동작:**
1. 햄버거 탭 → 좌측에서 슬라이드 인 (300ms ease-out)
2. 배경 반투명 딤(`rgba(0,0,0,0.4)`) 표시
3. 딤 탭 or 우측 스와이프 → 닫힘
4. Drawer 너비: `min(80vw, 320px)`

```
┌──────────────────────┬───────────┐
│ Drawer                │ 딤 배경  │
│                       │ (탭하면  │
│ [아바타] 김대윤        │  닫힘)   │
│ ──────────────────    │           │
│ 🔍 검색               │           │
│                       │           │
│ 📁 프로젝트           │           │
│  ○ 전체               │           │
│  ○ 만화웹툰2026       │           │
│                       │           │
│ 👤 담당자             │           │
│  ○ 전체               │           │
│                       │           │
│ [필터 초기화]          │           │
└──────────────────────┴───────────┘
```

**Drawer 내부 구성 (위→아래):**
1. 사용자 아바타 + 이름 + 이메일 (헤더에서 이동)
2. 검색바 (인라인)
3. 프로젝트 필터
4. 담당자 필터
5. 우선순위 필터
6. 상태 필터
7. 즐겨찾기 필터
8. 필터 초기화 버튼
9. 하단: 설정·로그아웃

### D. 리스트 뷰 (모바일)

**현재(웹):** 테이블 형태 — 열: 업무명·담당자·마감·우선순위·상태·진행률 등

**모바일:** 카드 목록으로 전환

```
┌─────────────────────────────────────┐
│  🔍 검색                    [필터▾] │  ← 검색 + 필터 버튼 한 줄
│  [프로젝트:만화웹툰] [담당자:김대윤] │  ← 활성 필터 칩 (가로 스크롤)
├─────────────────────────────────────┤
│ ┌───────────────────────────────┐   │
│ │ [만화웹툰2026]  ●긴급  진행중  │   │ ← 카드 상단: 프로젝트·우선순위·상태
│ │ 바이어 문의 매일 확인하기      │   │ ← 업무명 (최대 2줄)
│ │ 박정찬  ·  D-3   ·  75%      │   │ ← 담당자·마감·진행률
│ └───────────────────────────────┘   │
│ ┌───────────────────────────────┐   │
│ │ [디자인]       ●보통  대기     │   │
│ │ 시안 3개 긴급 처리             │   │
│ │ 김현지  ·  D-8   ·  0%       │   │
│ └───────────────────────────────┘   │
└─────────────────────────────────────┘
```

**카드 탭 동작:**
- 탭 → 업무 상세 바텀 시트 열림 (편집 포함)
- 스와이프 좌 → "완료" 초록 배경 노출 → 완료 처리
- 스와이프 우 → "삭제" 빨간 배경 노출 → 삭제 확인

**필터 버튼 [필터▾]:**
- 탭 → 필터 바텀 시트 열림
- 활성 필터 수 뱃지 표시: `[필터 ②▾]`

**정렬:**
- 필터 바텀 시트 내부에 정렬 옵션 포함 (마감순·우선순위순·등록순)

**완료 항목:**
- 목록 최하단에 "완료 N건 ▼" 접힘 형태 (웹과 동일 패턴)

### E. 업무 추가 — FAB + 바텀 시트 (모바일)

**현재(웹):** 리스트 하단 "직접 입력 / AI 분석" 테이블 형 섹션

**모바일:** FAB → 바텀 시트

```
                                  ┌─────┐
                                  │  +  │ ← FAB (우하단 고정, 56px)
                                  └─────┘
```

**FAB 탭 시 → 바텀 시트 (슬라이드 업)**

```
┌─────────────────────────────────────┐
│  ▬  (드래그 핸들)                    │
│  새 업무 추가                        │
│  [직접 입력]  [AI 분석]              │ ← 탭 전환
│ ─────────────────────────────────── │
│  업무명 *                            │
│  [                               ]  │
│                                     │
│  담당자          마감일              │
│  [드롭다운 ▾]   [날짜 선택 ▾]       │
│                                     │
│  우선순위        상태               │
│  [드롭다운 ▾]   [드롭다운 ▾]        │
│                                     │
│  프로젝트                           │
│  [드롭다운 ▾]                       │
│                                     │
│  상세내용 ▾ (접힘, 탭하면 펼침)      │
│                                     │
│  [          저장하기          ]      │ ← 전체 너비 버튼
└─────────────────────────────────────┘
```

**AI 분석 탭:**
```
│  회의 내용을 붙여넣으세요...          │
│  [파일 첨부  +]                      │
│                                     │
│  [      AI로 분석하기       ]        │
```

**드롭다운 선택:**
- 탭 시 중첩 바텀 시트(또는 같은 바텀 시트 내 목록 교체)로 표시
- 웹의 DropPanel 팝업 대신

### F. 칸반 뷰 (모바일)

**현재(웹):** 4컬럼 나란히 (grid auto-fit, 컬럼당 최소 260px)

**모바일:** 한 번에 1컬럼, 스와이프로 이동

```
┌─────────────────────────────────────┐
│  대기 (3)  진행중  검토  완료        │ ← 상단 탭 (컬럼 선택)
│  ━━━━━━                             │ ← 현재 컬럼 언더라인
├─────────────────────────────────────┤
│                                     │
│  [카드] 바이어 문의 확인             │
│  [카드] 시안 3개 긴급 처리           │
│  [카드] 보고서 작성                  │
│                                     │
│                      [ + 업무 추가] │
└─────────────────────────────────────┘
```

**컬럼 전환:**
- 상단 탭 탭: 해당 컬럼으로 이동
- 좌우 스와이프: 컬럼 이동 (페이징)
- 현재 컬럼 위치: 탭 언더라인 + 인디케이터 점

**카드 인터랙션:**
- 탭 → 업무 상세 바텀 시트
- 롱프레스(500ms) → 드래그 모드 진입 (다른 컬럼으로 이동 가능)
- 드래그 시 다른 컬럼 탭으로 이동하면 해당 컬럼으로 드롭

### G. 캘린더 뷰 (모바일)

**현재(웹):** 3개월 세로 스크롤 (월간 그리드)

**모바일:** 월간 기본 + 주간 전환 가능

```
┌─────────────────────────────────────┐
│  ◀  2026년 3월  ▶           [주간] │
├────┬────┬────┬────┬────┬────┬──────┤
│ 월  │ 화  │ 수  │ 목  │ 금  │ 토  │ 일  │
├────┼────┼────┼────┼────┼────┼──────┤
│    │    │    │    │    │    │      │
│  3  │  4  │  5  │  6  │  7  │  8  │  9  │
│ ●  │    │ ●● │    │ ●  │    │      │ ← 도트(업무 수)
├────┼────┼────┼────┼────┼────┼──────┤
```

**날짜 셀 크기:**
- 모바일에서는 이벤트 칩 텍스트 표시 불가 → **도트(●)로 표시**
- 도트 색상: 우선순위별 (긴급=빨강, 보통=주황, 낮음=회색)
- 도트 최대 3개, 초과 시 "+N" 텍스트

**날짜 탭:**
- 탭한 날짜 하이라이트
- 화면 하단에 해당 날짜 업무 목록 슬라이드 업 (절반 높이 바텀 시트)

```
┌─────────────────────────────────────┐
│  [캘린더 그리드 - 상단 절반]         │
├─────────────────────────────────────┤
│  3월 5일 (수) 업무 2건              │ ← 바텀 시트 헤더
│  ─────────────────────────────────  │
│  ● 바이어 문의 확인  진행중          │
│  ● 시안 3개 처리    긴급            │
│                                     │
│  [+ 이 날짜에 업무 추가]            │
└─────────────────────────────────────┘
```

**스와이프:**
- 좌우 스와이프: 월 이동

### H. 대시보드 (모바일)

**현재(웹):** KPI 카드 4개 + 팀원별 표 + 진행률 바

**모바일:** 세로 스크롤 단일 열

```
┌─────────────────────────────────────┐
│  전체   진행중   완료    지연        │ ← KPI 4개 가로 배치 (각 25%)
│   24     8      12       4          │
├─────────────────────────────────────┤
│  [이번 주 ▾]                        │ ← 기간 필터
├─────────────────────────────────────┤
│  ▣ 팀원별 현황                      │
│  김대윤  ██████░░  6/8건            │
│  박정찬  ████░░░░  4/8건            │
│  김현지  ██░░░░░░  2/8건            │
├─────────────────────────────────────┤
│  ▣ 프로젝트별 현황                  │
│  만화웹툰2026  ████████░░  80%      │
│  디자인        █████░░░░░  50%      │
└─────────────────────────────────────┘
```

**KPI 카드 탭:**
- 탭 시 해당 조건으로 리스트 뷰 이동 + 필터 자동 적용 (웹과 동일)

**팀원별 현황 표:**
- 가로 스크롤 테이블 대신 → 진행률 바 카드 형태로 변환

**모바일:** 전체 높이 바텀 시트

```
┌─────────────────────────────────────┐
│ ▬                          [저장]  │ ← 드래그 핸들 + 우상단 저장

│                                     │
│  담당자       마감일                │
│  [박정찬 ▾]  [3/5 수 ▾]           │
│                                     │
│  프로젝트                           │
│  [만화웹툰2026 ▾]                  │

### J. 필터 — 바텀 시트 (모바일)

**현재(웹):** 좌측 사이드바 항상 표시

**모바일:** [필터▾] 버튼 탭 → 바텀 시트

```
┌─────────────────────────────────────┐
│ ▬             필터           [초기화]│
│ ─────────────────────────────────── │
│  프로젝트                           │
│  [전체] [만화웹툰2026] [디자인]      │ ← 가로 스크롤 칩
│                                     │
│  담당자                             │
│  [전체] [김대윤] [박정찬] [김현지]  │
│                                     │
│  우선순위                           │
│  [전체] [긴급] [보통] [낮음]        │
│                                     │
│  상태                               │
│  [전체] [대기] [진행중] [검토] [완료]│

│  정렬                               │
│  [마감순] [우선순위순] [등록순]      │

## 6. 공통 설계 원칙 (이 앱 기준)

### 터치 타깃
- 모든 탭 가능 요소: **최소 44×44px** (시각 크기가 작아도 padding으로 확보)
- 리스트 카드 행 높이: **최소 64px**
- 하단 탭 바 각 탭: **전체 너비 ÷ 탭 수** (보통 64~72px)

### 폰트 크기
- 최소 **12px** (현재 9px 반복 배지는 모바일에서 11px로 상향)
- 입력 필드 **16px** 이상 (iOS 자동 줌 방지)
- 카드 업무명: **14px**, 보조 정보: **12px**

### 여백
- 카드 padding: **14px 16px**
- 카드 간 gap: **8px**
- 화면 좌우 padding: **16px**
- 바텀 시트 내부 padding: **20px 16px**

### 애니메이션
- Drawer 슬라이드: `300ms cubic-bezier(0.32, 0.72, 0, 1)`
- 바텀 시트 슬라이드 업: `350ms cubic-bezier(0.32, 0.72, 0, 1)`
- 딤 페이드인: `200ms ease`
- 스와이프 복귀: `200ms ease`

### Safe Area
```css
/* 모든 하단 고정 요소에 적용 */
padding-bottom: calc(16px + env(safe-area-inset-bottom));
```
